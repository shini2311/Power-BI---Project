<h1>HR Analytics Project Report</h1>

<b>Project Type: </b> HR Analytics & Dashboarding

<b>Tool Used: </b> Power BI

<h2> 1. Project Objective</h2>
  The goal of this project was to analyze the company’s HR recruitment process and provide insights that help improve hiring efficiency, recruiter performance, and candidate movement across different stages such as Submitted, Shortlisted, Interviewed, Selected, and Rejected.

<h2>2. Key Ojectives</h2>

<ol>

    • Understand applicant flow from Submitted → Shortlisted → Interviewed → Selected → Rejected

    • Evaluate recruiter-wise performance

    • Analyze client hiring trends

    • Identify delays in the hiring process

    • Track time taken between each recruitment stage

    • Provide an interactive Power BI dashboard for monitoring
    
</ol>

<h2>3. Dataset Overview</h2>


  The dataset provided included the following key fields:
<ol>

 • HR Team Lead

 • Recruiter Name

 • Client Name

 • Job Role

 • Applicant Name

 • Application Submitted On

 • Status Updated On

 • Current Application Status (Submited, Shortlisted, Interviewed, Selected, Rejected)

</ol>
<b>Total records: </b>1030

<h2>4. Data Cleaning & Processing</h2>

<ol> 

 • Removed duplicates
 
 • Standardized date formats

 • Cleaning inconsistent status values

 • Filled missing values where applicable
 
 </ol>
 

<h2>5. Insights & Findings</h2>

<ol>

  1️⃣ <b>Hiring Funnel Performance</b>

<ol>
  
  • A significant drop occurs from Shortlisted → Interviewed, indicating lack of engagement or follow-up from some recruiters.

  • The overall selection rate remains moderate, suggesting a balanced but improvable talent quality.

</ol>

2️⃣ <b>Recruiter Productivity</b>

<ol>
  
  • Some recruiters handle significantly more applications than others, showing workload imbalance.

  • Recruiters with higher touchpoints showed better conversion rates.

</ol>

3️⃣ <b>Client-Level Insights</b>

<ol>

  • Certain clients have high rejection rates, indicating mismatched job roles or strict screening criteria.

  • Consistent hiring seen for specific job roles (e.g., Data Analyst, HR Executive, Sales Coordinator).
</ol>

4️⃣<b> Time-to-Hire</b>

<ol>

  • Average time-to-hire varies widely across recruiters.

  • Delays occur mostly at the Interview Scheduling stage, impacting candidate experience.
</ol>
</ol>
<h2>6. Dashboard Summary</h2>

The interactive dashboard contains:
<ol>

  • Overall Recruitment KPIs

  • Recruiter Wise Performance Analysis

  • Monthly Hiring Trend

  • Client-wise Applicant Status

  • Funnel Visual – From Submitted to Selected

  • Daily Application Flow
</ol>
The dashboard helps HR managers and team leads monitor the entire process in real time and make data-driven decisions.

<h2>7. Recommendations</h2>

1️⃣Optimize recruiter allocation

Distribute applicants evenly among recruiters to avoid overload and delays.

2️⃣ Improve interview follow-ups

Set automated reminders to reduce drop-offs between shortlisting and interviews.

3️⃣ Client screening alignment

Review expectations with clients having high rejection rates.

4️⃣ Track KPIs monthly

A monthly performance dashboard will improve transparency and accountability.

<h2>8. Conclusion</h2>

This HR Analytics project delivers a complete view of the recruitment journey, improves transparency, and supports better decision-making.
It helps HR teams optimize hiring processes, reduce bottlenecks, and improve recruiter accountability.
